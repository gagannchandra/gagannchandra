### 🔹 [HealthAI — Symptom-Based Disease Prediction](https://github.com/gagannchandra/healthai-disease-prediction) · [Live ↗](https://healthai-disease-prediction.vercel.app/)
> *Python · FastAPI · Scikit-learn · React 19 · Tailwind CSS · Render*
> B.Tech Final Year Project (AI Specialization)

Built an end-to-end ML application that predicts diseases from user-reported symptoms. The core question was whether a soft-voting ensemble across three heterogeneous classifiers adds meaningful robustness over any single model — it does, but not dramatically, which was itself useful to learn.

- **87.84% ensemble accuracy** across 141 disease classes and 343 symptom features on a 102,238-sample dataset — soft-voting over Decision Tree, Random Forest, and Naive Bayes averaged `predict_proba` outputs
- Returns top-3 ranked diagnoses with confidence scores, recommended specialist, medications, diet, and precautions
- Voice input via Web Speech API; Recharts model-comparison chart; per-user prediction history
- FastAPI + Pydantic v2 backend; ~5.3 ms average response time at ~187 req/sec per core

### 🔹 [GitHub Repository Health Checker](https://github.com/gagannchandra/github-health-checker) · [Live ↗](https://github-health-checker-ktmb.onrender.com/) · [Demo ↗](https://youtu.be/M27N7L4T-bE)
> *Python · FastAPI · GitHub REST API · NVIDIA LLaMA 3.1 · Uvicorn*

Takes any public `owner/repo`, fetches live metrics concurrently from the GitHub API, and sends them to NVIDIA-hosted **LLaMA 3.1 (8B Instruct)** to generate a plain-English health report with strengths, concerns, and a score out of 100.

- Concurrent GitHub API calls for stars, forks, open issues, contributors, last commit date, license, and repo size
- Structured LLM prompt engineering → output: strengths · concerns · overall summary · health score
- Auto-assigns a status badge: **Active / Inactive / Needs Review**
- Async FastAPI + Uvicorn backend; dark UI with metric cards; deployed on Render

### 🔹 [TaskFlow — Team Task Manager](https://github.com/gagannchandra/team-task-manager) · [Live ↗](https://team-task-managerr.up.railway.app)
> *React 19 · Node.js · Express · MongoDB Atlas · JWT · Railway*

Full-stack collaborative task management app with role-based access control. Deployed as a monorepo single-service — Express serves both the API and the React build in production, no separate frontend deployment needed.

- JWT auth with bcrypt; role-based middleware guards — Admin (full CRUD) vs Member (view & update assigned tasks)
- Kanban-style task board; dashboard analytics with task status breakdown and team overview
- Mongoose models: User, Project, ProjectMember, Task; `express-validator` input validation
- Monorepo deploy: root `postinstall` cascades to sub-packages; Express serves `frontend/dist/` statically in production

### 🔹 [Shortly — URL Shortener with Analytics](https://github.com/gagannchandra/shortly-url-shortener) · [Live ↗](https://gagannchandra.pythonanywhere.com/)
